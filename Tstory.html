웹페이지 코딩 백서: 2트랙 AI 협업 방법론

트랙 A: Tistory 경량 · 트랙 B: GitHub Pages 풀옵션 — JS는 FAQ 아코디언과 복사 버튼만 사용

1. 목적

  • 플랫폼 제약과 목표 성격에 맞춰 2트랙(two-track)으로 개발·배포.
  • 트랙 A: Tistory(티스토리) 경량·반응형·즉시 붙여넣기.
  • 트랙 B: GitHub Pages(깃허브 페이지) 풀옵션(JavaScript, JS)·고급 인터랙션.
  • ChatGPT와 Claude의 강점 분업을 통해 속도·안정성·확장성 동시 확보.

2. 트랙 구분

트랙 A — Tistory 경량 버전

  • 환경: 제한적 <script> 허용, 외부 로더 제약 → 경량 구조 필수.
  • 원칙: HTML/CSS 중심, JS는 FAQ 아코디언과 복사 버튼만.
  • 모바일: 320px 기준 반응형, 폰트·이미지 최소화.
  • 전문용어 병기: 접근성(Accessibility, a11y)·검색최적화(Search Engine Optimization, SEO).
  • 담당 권장: ChatGPT (경량 설계·호환성·즉시 실행에 최적).

트랙 B — GitHub Pages 풀옵션 버전

  • 환경: 자유로운 JS/라이브러리, 정적 배포 + 글로벌 CDN(Content Delivery Network, CDN).
  • 원칙: 리치 인터랙션, SPA(Single Page Application) 네비게이션, API(Application Programming Interface) 연동.
  • 확장: 차트/다이어그램, 애니메이션, 폼 처리, 결제/후원 위젯 가능.
  • 담당 권장: Claude (모듈 JS·시각효과·대시보드·연동에 강점).

3. 협업 방법론

공통 콘텐츠

  • 백서 본문: 철학·아키텍처·운영 원칙·마케팅·CRM·실행 로드맵·KPI·핵심 메시지.
  • 톤: 광고·댓글·추천 제거 → 정적 게스트(Static guest) 몰입.

차별 구현

  • 트랙 A: CSS 인포그래픽(도넛/바 대신 바 형태), 아코디언, 복사 버튼.
  • 트랙 B: 차트 라이브러리, 스크롤 이펙트, SPA 라우팅, API 기반 대시보드.

검수·배포

  • Tistory: 단일 HTML 파일로 붙여넣기 → 즉시 발행.
  • GitHub Pages: 저장소 푸시 → Pages 활성화 → 커스텀 도메인 + HTTPS.

4. 실행 로드맵

  1. Day 1: 공통 콘텐츠 확정(본문/요약/메타).
  2. Day 2: 트랙 A(Tistory) 시안 완성·발행.
  3. Day 3: 트랙 B(GitHub Pages) 풀옵션 시안 완성.
  4. Day 4: 성능/접근성 점검(a11y, 320px, 대비).
  5. Day 5: 병행 배포 및 입구(YouTube/News/Brunch) 연결.

5. KPI(텍스트 중심 + CSS 바 시각화)

  • 3개월: 월 방문자 1,000 / 구독 50 / 후원자 5
  • 6개월: 월 방문자 5,000 / 구독 200 / 후원자 20
  • 12개월: 월 방문자 10,000 / 구독 500 / 후원자 50
철학적 일관성
실행 가능성
비용 효율성

정성지표는 메일 답장·재방문 시간 등 내측 지표로 보완(알고리즘 의존 최소화).

6. 핵심 성공 포인트

  • 환경 맞춤: 제약은 단순화, 자유는 확장으로 전환.
  • AI 분업: ChatGPT(경량·안정) + Claude(풀JS·연동).
  • 정적 게스트 UX: 한 페이지 한 메시지·줄 세우기(flow control).
  • 후원=연료 모델: 광고·댓글·추천 제거와 공존.

7. 최종 제안

  • 본사이트(Tistory) 운영: ChatGPT 방식으로 경량·무소음 유지.
  • 프로젝트/이벤트(GitHub Pages): Claude 방식으로 리치 인터랙션.
  • 콘텐츠는 동일, 기능만 차등 적용 → 관리 용이·브랜드 일관.

8. 문답(FAQ) — 아코디언

  • 플랫폼 제약 및 차단 가능성.
  • 모바일 성능·접근성(a11y) 유지.
  • 정적 게스트 철학과의 합치.
  • 차트·애니메이션·SPA 라우팅.
  • API 연동·폼 처리·웹앱 전환.
  • CDN·HTTPS·자동 배포.
  • 본문·메시지·KPI는 동일 원본에서 파생.
  • 트랙 A/B는 표현 레이어만 차등.
  • 버전 관리 문서로 변경 이력 동기화.
